# Scenario 1.2 실습 가이드: 커밋 메시지 수정 및 쪼개기/합치기

> **상황**: PR을 올리려고 커밋 히스토리를 보니 `wip`, `fix typo`, `진짜 수정`, `아 또 수정`...
> 이대로 올리면 리뷰어가 한숨을 쉴 것입니다. 깔끔하게 정리하세요!

### 📋 해야 할 일

setup-script.sh를 실행하여 실습 환경을 구성하세요.

### ✅ 확인 방법

- `git log --oneline` 실행 시 6개의 커밋이 보이나요?
- `wip`, `fix typo`, `진짜 수정`, `아 또 수정` 커밋이 보이나요?

## 🎯 미션 1: 현재 히스토리와 변경 내용 파악하기

### 📋 해야 할 일

1. 커밋 히스토리를 확인하세요.
2. 각 커밋이 어떤 파일을 수정했는지 파악하세요.
3. 어떤 커밋끼리 합쳐야 하는지 판단하세요.

### 💡 힌트

- 커밋별 변경 파일 목록까지 보기: `git log --oneline --____`
- 특정 커밋의 코드 변경 내용 보기: `git show <커밋해시>`
- `wip` ~ `아 또 수정` 4개 커밋이 모두 같은 기능(`cart.js`)을 수정하고 있음을 확인하세요.

### ✅ 확인 방법

- 커밋 3~6이 모두 `cart.js`를 수정하고 있나요?
- 커밋 2(`feat: 장바구니 UI 추가`)는 깔끔한 커밋이라 건드리지 않아도 된다고 판단했나요?

## 🎯 미션 2: Interactive Rebase로 커밋 합치기 (핵심!)

### 📋 해야 할 일

`git rebase -i`를 사용하여 지저분한 4개 커밋(`wip` ~ `아 또 수정`)을 **의미 있는 1개 커밋**으로 합치세요.

### 💡 힌트

- 인터랙티브 리베이스 시작: `git ______ -i HEAD~?`
  - `?`에는 합칠 커밋 수(4개)를 넣으세요
- 에디터가 열리면 커밋 목록이 표시됩니다
- 첫 번째 커밋은 `pick`으로 유지하고, 나머지를 `s______`로 변경하세요
  - 이 키워드는 "바로 위 커밋에 합치기"라는 뜻입니다
- 첫 번째 줄을 squash로 바꾸면 에러! 반드시 pick이 하나 있어야 합니다

### 에디터 조작법

- **vim**: `i` (편집모드) → 수정 → `Esc` → `:wq` (저장 종료)
- **nano**: 바로 수정 → `Ctrl+X` → `Y` → `Enter`
- 에디터를 바꾸고 싶다면: `git config --global core.editor "nano"`

### ✅ 확인 방법

- `git log --oneline`에서 커밋이 **3개**로 줄었나요?
- `git show HEAD --stat`으로 합쳐진 커밋의 변경 파일을 확인했나요?
- `cat cart.js`로 최종 코드가 정상인지 확인했나요? (addToCart, removeFromCart, renderCart 함수 모두 포함)

## 🎯 미션 3: 합쳐진 커밋 메시지 의미 있게 작성하기

### 📋 해야 할 일

미션 2에서 squash 후 열리는 두 번째 에디터에서, 의미 있는 커밋 메시지를 작성하세요.

### 💡 힌트

- 두 번째 에디터에는 합쳐진 커밋들의 기존 메시지가 모두 나열됩니다
- 기존 메시지를 모두 지우고, Conventional Commits 형식으로 새 메시지를 작성하세요
- 형식: `feat: 기능 설명` (첫 줄) + 빈 줄 + 상세 설명 (선택)
- 예시:

  ```
  feat: 장바구니 기능 구현

  - 상품 추가/삭제
  - 수량 관리
  ```

### ✅ 확인 방법

- `git log --oneline`에서 새 커밋 메시지가 보이나요?
- `git log -1`로 상세 메시지까지 확인했나요?

## 🎯 미션 4: (심화) reword로 메시지만 수정하기

### 📋 해야 할 일

"feat: 장바구니 UI 추가" 커밋의 메시지를 더 구체적으로 수정해보세요.
(코드 변경 없이 메시지만 수정합니다)

### 💡 힌트

- `git rebase -i HEAD~?`로 시작하세요 (수정할 커밋을 포함하는 범위)
- 해당 커밋의 `pick`을 `r______`로 변경하세요
- 이 키워드는 "커밋 유지, 메시지만 수정"이라는 뜻입니다
- 저장하면 메시지 편집 에디터가 열립니다

### ✅ 확인 방법

- `git log --oneline`에서 메시지가 변경되었나요?
- 커밋 해시가 변경된 것을 확인했나요? (리베이스는 새 커밋을 만듭니다)

---

## 🎯 미션 5: (심화) 실수했을 때 리베이스 취소하기

### 📋 해야 할 일

리베이스를 시작한 후, 도중에 취소하여 원래 상태로 돌아가보세요.

### 💡 힌트

- 리베이스 도중 취소: `git rebase --_____`
- 이 명령어는 리베이스 시작 전 상태로 완벽하게 복구합니다

### ✅ 확인 방법

- 취소 후 `git log --oneline`이 리베이스 시작 전과 동일한가요?
